# Инлайн: PDF в чат по дате

Чтобы при вводе **@raspisaniecpsu_bot 13.02.2026** (или «сегодня» / «завтра») в чат приходил **PDF**, а не кнопка, нужно выполнить три условия.

---

## 1. В .env на сервере задан HTTPS-адрес

Файл: `/home/bot_rasp/.env`

```env
PUBLIC_BASE_URL=https://bot.sebog1.ru
```

- Без `https://` инлайн не отправит документ, только кнопку.
- Без слеша в конце.
- После правки: `sudo systemctl restart bot_rasp`.

---

## 2. Сайт отдаёт PDF по ссылке

В браузере откройте (подставьте нужную дату):

```
https://bot.sebog1.ru/schedule/2026-02-13.pdf
```

- Должен открыться или скачаться PDF.
- Если 404 — проверьте: запущен ли `bot_rasp_web`, настроен ли Nginx (см. NGINX_SETUP.md).

---

## 3. В архиве есть расписание на эту дату

PDF в инлайне отдаётся только по тем датам, для которых расписание уже сохранено (пришло на почту и было обработано, или загружено вручную). Для дат без расписания бот покажет кнопку «Открыть в боте».

---

## Проверка по логам

На сервере после запроса `@raspisaniecpsu_bot 13.02.2026`:

```bash
sudo journalctl -u bot_rasp -n 20 --no-pager
```

- **`Inline: отправка PDF в чат для 13.02.2026`** — бот отдаёт документ, в чат должен прийти PDF.
- **`Inline: PUBLIC_BASE_URL не задан или не https`** — в .env нет подходящего `PUBLIC_BASE_URL`.
- **`Inline: на 13.02.2026 расписания нет в архиве`** — на эту дату нет сохранённого расписания.
- **`answer_inline_query failed: ...`** — Telegram отклонил ответ. Часто из‑за того, что **не может забрать PDF по ссылке** (404, таймаут, редирект на HTTP, ошибка SSL).

---

## Сайт доступен, но документ в чат не приходит

Если в браузере `https://bot.sebog1.ru/schedule/2026-02-13.pdf` открывается, а в инлайне приходит «Нажмите, чтобы открыть бота»:

1. **Посмотреть точную ошибку в логах:**
   ```bash
   sudo journalctl -u bot_rasp -n 30 --no-pager | grep -A2 "answer_inline_query failed"
   ```
   Если в тексте ошибки есть *url*, *document*, *HTTP* — Telegram не смог скачать файл по ссылке.

2. **Проверить URL с сервера (как это делает Telegram):**
   ```bash
   curl -I "https://bot.sebog1.ru/schedule/2026-02-13.pdf"
   ```
   Ожидается: **200 OK**, заголовок **Content-Type: application/pdf**. Не должно быть редиректа (301/302) на другой адрес или на HTTP.

3. **Проверить, что бот на сервере видит тот же PUBLIC_BASE_URL:**
   ```bash
   grep PUBLIC_BASE_URL /home/bot_rasp/.env
   ```
   Должно быть: `PUBLIC_BASE_URL=https://bot.sebog1.ru` (именно **https**).

4. **Убедиться, что расписание на эту дату есть в базе** (панель или письмо с вложением уже обработаны). Без записи в БД маршрут `/schedule/...pdf` отдаёт 404.
