# Идеи улучшений бота расписания

Краткий список возможных доработок по разделам. Можно выбирать по приоритету.

---

## Бот (Telegram)

| Улучшение | Описание | Сложность |
|-----------|----------|-----------|
| **Список дат с кнопками** | Команда `/list` показывает инлайн-кнопки с датами (например 5 последних). Нажал — пришёл PDF на выбранную дату. | Низкая |
| **Напоминание перед парой** | Пользователь пишет «напомни за 10 минут до первой пары» — бот сохраняет настройку и в нужный день отправляет сообщение. Нужна таблица напоминаний и планировщик (APScheduler уже есть). | Средняя |
| **Избранная группа** | Сохранить у пользователя группу (например 2ИС24Б). Кнопка «Моё расписание» отправляет расписание на сегодня/завтра для этой группы. Нужна таблица `user_favorites` и возврат поиска по группе. | Средняя |
| **Голосовые сообщения** | Обработка голосовых: распознать «расписание на завтра» и ответить PDF (Speech-to-Text API или локальная модель). | Высокая |
| **Онбординг при /start** | Первый запуск: 2–3 шага (подписка на канал → «Включить напоминания?» → «Указать группу для быстрого доступа?»). | Низкая |
| **Кнопка «Поделиться ботом»** | В меню или при /start — инлайн-кнопка «Поделиться» с готовой ссылкой на бота. | Низкая |

## Веб-панель

| Улучшение | Описание | Сложность |
|-----------|----------|-----------|
| **Страница «Расписания»** | Список всех дат с PDF: скачать, удалить устаревшее. API: `GET /api/schedules`, `DELETE /api/schedules/<date>`. | Низкая |
| **График активности** | На дашборде: подписчики за 7/30 дней, активность по дням (по `last_activity`). | Низкая |
| **Объявление в боте** | В панели: поле «Текст объявления» + «Опубликовать». Бот при следующем /start или отдельной командой показывает это сообщение (или закрепляет в группе). | Средняя |
| **Экспорт обратной связи** | Кнопка «Скачать CSV» на странице обратной связи — выгрузка всех обращений с датами и ответами. | Низкая |
| **Смена пароля** | В панели: «Настройки» → смена пароля (хранить хэш в .env или в БД). | Низкая |

## Расписание и данные

| Улучшение | Описание | Сложность |
|-----------|----------|-----------|
| **Вернуть поиск по группе/преподавателю** | Снова показывать текстовое расписание по запросу «2ИС24Б» или «Петренко» (парсер и таблица уже есть). Опционально — сначала кнопки «Сегодня»/«Завтра», потом ответ. | Низкая |
| **Экспорт в календарь (.ics)** | По группе или дате: генерация .ics файла «Расписание на неделю» для импорта в Google Calendar / Outlook. | Средняя |
| **Уведомление «расписание на завтра выложено»** | Раз в день (например в 18:00) проверять: есть ли расписание на завтра; если да — разослать подписчикам короткое сообщение «Расписание на завтра уже в боте». | Низкая |
| **Проверка дубликатов при загрузке с почты** | Перед сохранением проверять: не загружено ли уже расписание на эту дату (по дате из письма/файла). Не перезаписывать без явного указания. | Низкая |

## API и производительность

| Улучшение | Описание | Сложность |
|-----------|----------|-----------|
| **Кэш для get_schedule** | В памяти кэшировать результат «есть ли PDF на дату» на 1–2 минуты, чтобы при частых инлайн-запросах не дергать БД каждый раз. | Низкая |
| **Health check** | Endpoint `GET /health` или `/ping`: возвращает 200 и версию/время работы. Удобно для мониторинга и UptimeRobot. | Низкая |
| **Лимит на /schedule/*.pdf** | Ограничение запросов по IP (например 60 в минуту), чтобы ссылку не качали пачкой. | Низкая |

## Инфраструктура и надёжность

| Улучшение | Описание | Сложность |
|-----------|----------|-----------|
| **Бэкап БД и архива** | Cron: раз в день копировать `data/bot.db` и папку `archive/` в бэкап-директорию или на облако (rclone, S3). | Низкая |
| **Docker Compose** | Один `docker-compose.yml`: бот, веб (api.py), nginx. Запуск одной командой, проще перенос на другой сервер. | Средняя |
| **Алерты при падении** | Скрипт или сервис: раз в 5 минут проверять, что бот и веб отвечают; при падении — отправить сообщение админу в Telegram. | Низкая |

---

## Что внедрить в первую очередь

- **Быстро и полезно:** список дат с кнопками (`/list` → кнопки → PDF), страница «Расписания» в панели, health check, бэкап по расписанию.
- **Средний приоритет:** напоминание перед парой, объявление в боте, уведомление «расписание на завтра выложено», кэш для get_schedule.
- **По желанию:** возврат поиска по группе/преподавателю, экспорт в .ics, Docker, онбординг.

Можно выбрать 2–3 пункта — реализую по шагам.
